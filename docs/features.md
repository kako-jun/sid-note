# 機能一覧

## 運指記譜

音符ごとに左手（押弦）と右手（ピッキング）の情報を記録できる。

**左手の記録項目:**
- 指番号（1〜4）
- 弦番号（1〜4弦）
- フレット番号（0〜24フレット）
- 種別（通常押弦 / ミュート / ゴーストノート / コード一括）
- ピッチ（例: A2, C＃3）
- 音程（例: 1, ♭3, 5, ♭7）

**右手の記録項目:**
- 弦番号
- ストロークの向き（ダウン / アップ / サムピック）
- ミュートする弦の一覧

## コード分解

コード名（例: Cmaj7, Am, G7）を入力すると、4弦ベース上のすべての弦・フレットの組み合わせを自動計算して表示する。

ルート音から半音インデックスを使って各構成音のフレット位置を求め、弦ごとのポジションに変換する。

**対応するコードタイプ:**
- メジャー / マイナー / ディミニッシュ / オーギュメント
- sus4
- メジャー7 / ドミナント7 / マイナー7 / オーギュメント7
- パワーコード（5th）
- オクターブユニゾン（8）
- ALL_KEYS（全12半音）/ WHITE_KEYS（白鍵のみ）

## スケール表示

キー（調）を指定すると以下の情報を表示する：

- スケール構成音（全24キー対応：メジャー12キー＋マイナー12キー）
- ダイアトニックコード（トライアド）
- ダイアトニックコード（7th）
- 五度圏上の位置（外円＝メジャーキー、内円＝マイナーキー）

## 和声分析

コードとキーを指定すると以下を表示する：

- 機能和声のディグリー（Ⅰ〜Ⅶ）
- 各ディグリーの説明（例: Ⅴ Dominant — 緊張・推進）
- ローマ数字表記（トライアド：Ⅰ / Ⅱm / Ⅶdim など）
- 7thコードのローマ数字表記（ⅠM7 / Ⅱm7 / Ⅶm7♭5 など）
- カデンツの種類（完全終止 / 変格終止 / 偽終止 / 半終止 / フリジアン終止）
- 各音のコードトーンラベル（例: Tonic Note, Dominant Note）

## 異名同音の正規化

C＃ と D♭ のように表記は異なるが音高が同じ音を正しく同一視する。

Rust 実装では半音インデックスで比較するため、正確に判定できる。
TypeScript 実装は文字列比較のため、この判定に既知のバグがある（詳細は `docs/todo.md` 参照）。

## WASM 高速化（準備中）

音楽理論計算のロジックを Rust で実装し、WebAssembly としてビルドする準備が整っている。

Phase 3 時点で Rust 側のすべての関数（26関数）の実装とテストが完了している。
Phase 4（Next.js からの WASM 呼び出し）は保留中。

Rust 実装の主な利点：
- 半音インデックスによる正確な異名同音判定
- コンパイル時の型検証
- 将来的な npm パッケージとしての切り出し
