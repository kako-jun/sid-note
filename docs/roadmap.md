**優先度: 高**

1. `wasm-pack` でビルドして `pkg/` ディレクトリを生成する
2. `src/lib/wasm-init.ts` を作成して初期化処理を一元管理する
3. まず `Keyboard.tsx` など 1 コンポーネントで試験統合する

## 6 弦対応・ギター対応

現在は 4 弦ベース（標準チューニング E1-A1-D2-G2）のみ対応。

**拡張候補:**
- 5 弦ベース（B0 弦を追加）
- 6 弦ギター（E2-A2-D3-G3-B3-E4）
- ウクレレ（G4-C4-E4-A4）
- カスタムチューニング（ユーザー定義）

`convertFretsToPositions()` の弦判定ロジックを楽器定義として抽象化することで対応できる。

## npm パッケージとしての切り出し

Rust 製音楽理論ライブラリ（`rust-music/`）を独立した npm パッケージとして公開する。

**想定する手順:**
1. `git subtree split` で `rust-music/` を独立リポジトリに分離
2. WASM ビルドの CI/CD を構築（GitHub Actions）
3. `kordweb` のように npm レジストリに公開
4. sid-note 本体はこのパッケージに依存する形に変更

**意義:**
- ベースやギターの音楽理論アプリを作る他の開発者が利用できる
- 日本語表記（全角 ＃・♭）や機能和声・カデンツ判定など、sid-note 固有の要件が汎用ライブラリとして活用できる
